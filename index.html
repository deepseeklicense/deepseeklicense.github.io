DeepSeek Lisence FAQ

中文版 / English Version

DeepSeek 系列开源模型自发布以来受到了技术社区的广泛关注，不少开发者基于DeepSeek模型进行了衍生开发或搭建了上层应用。在这个过程中，我们也陆续收到关于模型开源许可和使用限制的咨询，为了方便大家进一步了解我们的开源政策，消除大家在使用DeepSeek模型时的顾虑，本文就其中常见的疑问在此一并解答。

DeepSeek 开源许可简介

DeepSeek 针对代码仓库（Code Repository）和模型（Model）分别适用不同的许可证：

  • 代码：适用MIT许可证
  • 模型：适用自建开源许可证（许可证示例）

代码适用的MIT许可证为标准许可证，在此不再展开。模型适用的自建许可证则是在OpenRAIL许可证基础上修改形成的，具有适配性、开放性和负责任的特点：

  • 适配性，侧重于适配模型参数、权重而非代码的开源许可。
  • 开放性，宽松的开源条件和限制，促使开源成果更好地被共享、利用和分发，激发更多的创造力。
  • 负责任，倡导在许可中采用基于行为的使用限制，以降低人工智能技术带来的危害风险。

开源政策 FAQ

以下是开发者在使用本许可证中常见的问题和解答。

DeepSeek 开源模型是否可以用于商业目的？

可以。

DeepSeek 当前的开源模型均可用于包括商业目的在内的任何合法目的，包括但不限于直接部署使用，对模型进行衍生开发（微调、量化、蒸馏等）后部署使用，基于模型以及衍生模型开发自有产品后对外提供服务，或集成于模型平台进行分发或提供远程访问等。

使用 DeepSeek 开源模型，DeepSeek 是否会收取费用或者使用者的收益？

不会。

DeepSeek 开源模型完全免费，无任何许可费用。使用者基于 DeepSeek 开源模型开发的任何衍生模型、产品应用以及模型生成物的权利均归属于使用者，DeepSeek 不会主张使用者使用产生的任何收益、利益。

使用 DeepSeek 开源模型前，是否需要向 DeepSeek 进行申请或登记？

不需要。

开发者可以自由获取和使用 DeepSeek 开源模型，无需向 DeepSeek 进行任何申请或登记。

基于 DeepSeek 开源模型开发衍生模型或产品，分发时是否需要进行声明？

需要。

DeepSeek许可证要求遵守声明条件。建议使用者在其分发的衍生模型或发布的产品中，通过声明、产品协议或者其他恰当方式提供许可证副本。

基于DeepSeek 开源模型开发衍生模型后，是否需要开源该衍生模型？

不需要。

DeepSeek 许可证不同于 GPL 等“著佐权（copyleft）”许可证，不以必须开源衍生模型为条件，使用者可以自由选择开源或不开源。

对 DeepSeek 开源模型进行修改、开发后，衍生模型的分发是否也必须使用DeepSeek许可证？

不需要。

衍生模型或产品如果继续在开源社区分发，开发者可以自由选择适用的许可证。

使用 DeepSeek 开源模型有哪些限制？

DeepSeek许可证与当前主流开源模型许可证一致，需要避免用于非法或危险目的，具体见许可证Attachment A（Use Restrictions）部分。
